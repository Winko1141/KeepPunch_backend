# KeepPunch 项目总览

## 一、项目概述

**KeepPunch** 是一款专注于习惯养成和目标追踪的打卡应用。用户可以创建自定义任务，设置重复规则，并通过每日打卡来追踪自己的完成进度。应用提供了直观的周/月/年统计视图，帮助用户了解自己的习惯养成情况。

### 1.1 核心价值
- **习惯养成**：帮助用户建立并坚持良好的日常习惯
- **目标追踪**：可视化追踪任务完成情况，提供成就感
- **数据洞察**：通过统计分析了解自己的行为模式
- **提醒机制**：定时提醒防止遗忘

## 四、核心功能模块

### 4.1 任务管理

- **创建任务**：设置任务名称、图标、重复规则、目标次数
- **任务列表**：按日期和分类筛选任务
- **任务详情**：查看任务完整信息和统计数据
- **任务编辑**：修改任务属性
- **终止/恢复**：软删除机制

### 4.2 打卡功能

- **每日打卡**：对任务进行完成标记
- **取消打卡**：撤销当天的打卡操作
- **日期验证**：防止打卡未来日期

### 4.3 统计功能

- **周统计**：本周每日完成情况
- **月统计**：月度完成日历视图
- **年统计**：年度趋势分析
- **完成率**：总打卡次数、连续打卡天数、最长连续天数

### 4.4 分类管理

- **分类创建**：创建任务分类，设置颜色和图标
- **分类编辑**：修改分类属性
- **分类删除**：删除分类（关联任务变为未分类）
- **任务筛选**：按分类筛选任务列表

### 4.5 图标系统

- **系统图标**：预设图标库
- **自定义图标**：用户上传自定义图标
- **图标选择器**：创建任务时选择图标

### 4.6 通知提醒

- **单次提醒**：指定日期的一次性提醒
- **每日提醒**：每天固定时间提醒
- **每周提醒**：按周规则重复提醒
- **设备重启恢复**：重启后自动恢复所有提醒

## 五、数据库设计

### 5.1 主要数据表

#### task 表（任务）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| task_name | VARCHAR | 任务名称 |
| icon | VARCHAR | 图标标识 |
| repeat_type | INT | 重复类型（1=周重复, 2=周目标, 3=月目标） |
| week_rule | VARCHAR | 周重复规则（如 "1,3,5" 表示周一三五） |
| target_count | INT | 目标完成次数 |
| status | INT | 状态（1=活跃, 2=终止） |
| create_time | DATETIME | 创建时间 |
| reminder_time | VARCHAR | 提醒时间（HH:mm） |
| reminder_enabled | TINYINT | 是否启用提醒 |
| reminder_type | INT | 提醒类型（1=单次, 2=周, 3=日） |
| reminder_date | VARCHAR | 单次提醒日期 |
| reminder_week_rule | VARCHAR | 周提醒规则 |
| category_id | INT | 外键，关联分类 |

#### record 表（打卡记录）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |

| record_date | DATE | 打卡日期 |
| is_finish | TINYINT | 是否完成 |
| create_time | DATETIME | 创建时间 |

#### category 表（分类）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| name | VARCHAR | 分类名称 |
| icon | VARCHAR | 图标 |
| color | VARCHAR | 颜色值（HEX） |
| sort_order | INT | 排序顺序 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

#### icon 表（图标）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| name | VARCHAR | 图标名称 |
| display_name | VARCHAR | 显示名称 |
| icon_type | VARCHAR | 类型（system/custom） |
| icon_content | TEXT | 图标内容（base64） |
| icon_format | VARCHAR | 格式（png/jpg） |
| sort_order | INT | 排序 |
| is_active | TINYINT | 是否启用 |

#### task_count 表（任务计数）
| 字段 | 类型 | 说明 |

| id | INT | 主键 |

| month | INT | 月 |
| week | INT | 周 |
| total_count | INT | 完成次数 |

## 七、核心业务流程

### 7.1 任务创建与打卡流程

```
用户 → 创建任务 → 设置重复规则/提醒时间
        ↓
    保存到数据库
        ↓
    首页按日期显示任务
        ↓
用户点击打卡 → 验证日期有效性
        ↓
    检查是否已打卡
        ↓
    插入打卡记录
        ↓
    更新任务计数
        ↓
    返回成功
```

### 7.2 统计查询流程

```
用户选择时间范围（周/月/年）
        ↓
    前端发送请求
        ↓
    后端查询任务列表
        ↓
    后端查询打卡记录
        ↓
    组装统计数据
        ↓
    前端渲染视图
```

### 7.3 提醒通知流程

```
任务创建/更新时设置提醒
        ↓
    NotificationHelper 调度 Alarm
        ↓
    到达提醒时间 → AlarmReceiver 触发
        ↓
    构建并显示通知
        ↓
用户点击通知 → 打开应用
```

## 八、运行说明

### 8.1 后端启动

```bash
# 进入后端目录
cd KeepPunch_Backend

# 安装依赖
pnpm install

# 配置数据库
# 1. 创建数据库：keeppunch
# 2. 修改 db/index.js 中的数据库连接信息

# 启动开发服务器
pnpm dev

# 服务器运行在 http://localhost:3000
```

### 8.2 前端启动

```bash
# 进入前端目录
cd KeepPunch_frontend

# 修改后端地址
# 编辑 app/src/main/java/com/example/keeppunch/network/RetrofitClient.kt
# 将 BASE_URL 修改为你的后端地址

# 使用 Android Studio 打开项目并构建
# 或使用命令行
./gradlew assembleDebug
```
