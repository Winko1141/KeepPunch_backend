# 通知提醒模块

## 一、功能概述

通知提醒模块负责在指定时间提醒用户完成任务，主要功能包括：

- **定时提醒**：在指定时间推送通知
- **多种重复模式**：支持单次、每日、每周重复提醒
- **向后兼容**：兼容旧版本的提醒配置
- **精确唤醒**：使用 AlarmManager 精确唤醒设备
- **开机恢复**：设备重启后自动恢复提醒

## 二、实现原理

### 2.1 Android 定时机制

**AlarmManager 概述：**

Android 提供 AlarmManager 用于在未来的某个时间执行操作。即使应用未运行，定时任务也能触发。

**工作原理：**

```
应用注册定时任务
    ↓
AlarmManager 系统服务接管
    ↓
到达指定时间
    ↓
发送 PendingIntent
    ↓
触发 BroadcastReceiver
    ↓
显示 Notification
```

### 2.2 提醒类型设计

项目定义了三种提醒类型：

| reminder_type | 类型 | 调度策略 | 使用字段 |
|--------------|------|----------|----------|
| 1 | 单次提醒 | 指定日期时间触发一次 | reminder_date |
| 2 | 每周提醒 | 按指定周几重复 | reminder_week_rule |
| 3 | 每日提醒 | 每天固定时间触发 | 无 |

**提醒类型映射：**

```kotlin
when (reminderType) {
    1 -> scheduleOnceReminder(...)      // 单次
    2 -> scheduleReminderWeekly(...)    // 每周
    3 -> scheduleDailyReminder(...)     // 每日
    else -> scheduleBackwardCompatibleReminder(...)  // 兼容旧数据
}
```

### 2.3 向后兼容逻辑

**为什么需要兼容模式？**

早期版本可能没有 `reminder_type` 字段，或者使用不同的方式定义提醒规则。

**兼容策略：**

```kotlin
private fun scheduleBackwardCompatibleReminder(
    context: Context,
    task: TaskItem,
    alarmManager: AlarmManager,
    hour: Int,
    minute: Int
) {
    when (task.repeat_type) {
        1 -> scheduleWeeklyReminders(context, task, alarmManager, hour, minute)
        else -> scheduleDailyReminder(context, task, alarmManager, hour, minute)
    }
}
```

**兼容规则：**
- `repeat_type = 1`（周重复任务）→ 使用周提醒
- 其他 `repeat_type` → 使用每日提醒

### 2.4 RequestCode 生成策略

**问题：** 同一个任务可能有多个提醒（如每周一、三、五），如何区分？

**解决方案：**

```kotlin
private fun getRequestCode(taskId: Int, dayOfWeek: Int): Int {
    return taskId * 10 + dayOfWeek
}
```

**生成规则：**

| taskId | dayOfWeek | requestCode | 说明 |
|--------|-----------|-------------|------|
| 123 | 0 | 1230 | 每日提醒 |
| 123 | 1 | 1231 | 周一提醒 |
| 123 | 2 | 1232 | 周二提醒 |
| 123 | 7 | 1237 | 周日提醒 |

**设计原理：**
- `taskId * 10` 确保不同任务的 requestCode 不冲突
- `dayOfWeek` 范围 0-7，最多 8 个提醒
- 日提醒用 `dayOfWeek = 0`

**取消提醒：**

```kotlin
fun cancelTaskReminder(context: Context, taskId: Int) {
    val alarmManager = context.getSystemService(Context.ALARM_SERVICE) as AlarmManager
    val intent = Intent(context, AlarmReceiver::class.java)

    for (day in 0..7) {
        val requestCode = getRequestCode(taskId, day)
        val pendingIntent = PendingIntent.getBroadcast(
            context,
            requestCode,
            intent,
            PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
        )
        alarmManager.cancel(pendingIntent)
    }
}
```

### 2.5 精确唤醒机制

**问题：** Android 6.0+ 引入 Doze 模式，标准闹钟可能延迟。

**解决方案：**

```kotlin
if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
    alarmManager.setExactAndAllowWhileIdle(
        AlarmManager.RTC_WAKEUP,
        calendar.timeInMillis,
        pendingIntent
    )
} else {
    alarmManager.setExact(
        AlarmManager.RTC_WAKEUP,
        calendar.timeInMillis,
        pendingIntent
    )
}
```

**API 对比：**

| API | 最低版本 | 精确性 | Doze 模式 |
|-----|----------|--------|-----------|
| set() | API 1 | 不精确 | 延迟 |
| setExact() | API 19 | 精确 | 可能延迟 |
| setExactAndAllowWhileIdle() | API 23 | 精确 | 可唤醒 |

**所需权限：**

```xml
<uses-permission android:name="android.permission.SCHEDULE_EXACT_ALARM" />
<uses-permission android:name="android.permission.USE_EXACT_ALARM" />
```

### 2.6 通知渠道

**Android 8.0+ 要求：** 所有通知必须通过渠道发送。

**渠道创建：**

```kotlin
fun createNotificationChannel(context: Context) {
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
        val channel = NotificationChannel(
            CHANNEL_ID,
            CHANNEL_NAME,
            NotificationManager.IMPORTANCE_HIGH
        ).apply {
            description = CHANNEL_DESC
            enableVibration(true)
            enableLights(true)
        }

        val notificationManager = context.getSystemService(
            Context.NOTIFICATION_SERVICE
        ) as NotificationManager
        notificationManager.createNotificationChannel(channel)
    }
}
```

### 3.2 提醒调度入口

```kotlin
object NotificationHelper {

    const val CHANNEL_ID = "task_reminder_channel"
    const val CHANNEL_NAME = "任务提醒"
    const val CHANNEL_DESC = "定时提醒用户完成任务"

    const val EXTRA_TASK_ID = "extra_task_id"
    const val EXTRA_TASK_NAME = "extra_task_name"
    const val EXTRA_TASK_ICON = "extra_task_icon"

    fun scheduleTaskReminder(context: Context, task: TaskItem) {
        // 1. 检查提醒开关和时间
        if (!task.reminder_enabled || task.reminder_time.isNullOrEmpty()) {
            cancelTaskReminder(context, task.id)
            return
        }

        // 2. 解析时间（HH:mm）
        val timeParts = task.reminder_time.split(":")
        if (timeParts.size != 2) return
        val hour = timeParts[0].toIntOrNull() ?: return
        val minute = timeParts[1].toIntOrNull() ?: return

        // 3. 获取 AlarmManager
        val alarmManager = context.getSystemService(Context.ALARM_SERVICE) as AlarmManager

        // 4. 根据提醒类型调度
        val reminderType = task.reminder_type
        when (reminderType) {
            1 -> scheduleOnceReminder(context, task, alarmManager, hour, minute)
            2 -> scheduleReminderWeekly(context, task, alarmManager, hour, minute)
            3 -> scheduleDailyReminder(context, task, alarmManager, hour, minute)
            else -> scheduleBackwardCompatibleReminder(context, task, alarmManager, hour, minute)
        }
    }
}
```

### 3.3 单次提醒实现

```kotlin
private fun scheduleOnceReminder(
    context: Context,
    task: TaskItem,
    alarmManager: AlarmManager,
    hour: Int,
    minute: Int
) {
    val reminderDate = task.reminder_date ?: return

    val dateFormat = SimpleDateFormat("yyyy-MM-dd", Locale.getDefault())
    val targetDate = dateFormat.parse(reminderDate) ?: return

    val requestCode = getRequestCode(task.id, 0)
    val intent = createAlarmIntent(context, task)
    val pendingIntent = createPendingIntent(context, requestCode, intent)

    val calendar = Calendar.getInstance().apply {
        time = targetDate
        set(Calendar.HOUR_OF_DAY, hour)
        set(Calendar.MINUTE, minute)
        set(Calendar.SECOND, 0)
        set(Calendar.MILLISECOND, 0)

        // 已过期的提醒不设置
        if (timeInMillis <= System.currentTimeMillis()) {
            return
        }
    }

    setExactAlarm(alarmManager, calendar.timeInMillis, pendingIntent)
}
```

### 3.4 每周提醒实现

```kotlin
private fun scheduleReminderWeekly(
    context: Context,
    task: TaskItem,
    alarmManager: AlarmManager,
    hour: Int,
    minute: Int
) {
    val weekDays = if (task.reminder_week_rule.isNullOrEmpty()) {
        listOf(1, 2, 3, 4, 5, 6, 7)  // 默认每天
    } else {
        task.reminder_week_rule.split(",").map { it.toInt() }
    }

    weekDays.forEach { dayOfWeek ->
        val requestCode = getRequestCode(task.id, dayOfWeek)
        val intent = createAlarmIntent(context, task)
        val pendingIntent = createPendingIntent(context, requestCode, intent)

        val calendar = Calendar.getInstance().apply {
            set(Calendar.HOUR_OF_DAY, hour)
            set(Calendar.MINUTE, minute)
            set(Calendar.SECOND, 0)
            set(Calendar.MILLISECOND, 0)

            // 计算目标日期
            val currentDayOfWeek = when (get(Calendar.DAY_OF_WEEK)) {
                Calendar.SUNDAY -> 7
                else -> get(Calendar.DAY_OF_WEEK) - 1
            }

            var daysUntilNext = dayOfWeek - currentDayOfWeek
            if (daysUntilNext < 0) {
                daysUntilNext += 7
            } else if (daysUntilNext == 0 && timeInMillis <= System.currentTimeMillis()) {
                // 今天的时间已过，安排到下周
                daysUntilNext = 7
            }

            add(Calendar.DAY_OF_YEAR, daysUntilNext)
        }

        setExactAlarm(alarmManager, calendar.timeInMillis, pendingIntent)
    }
}
```

**周几计算说明：**

Java Calendar 的 DAY_OF_WEEK：
- SUNDAY = 1
- MONDAY = 2
- ...
- SATURDAY = 7

项目使用的周几表示（1-7 对应周一到周日）：
- 周一 = 1
- 周二 = 2
- ...
- 周日 = 7

**转换逻辑：**

```kotlin
val currentDayOfWeek = when (get(Calendar.DAY_OF_WEEK)) {
    Calendar.SUNDAY -> 7          // Calendar.SUNDAY=1 → 7
    else -> get(Calendar.DAY_OF_WEEK) - 1  // 其他减 1
}
```

### 3.5 每日提醒实现

```kotlin
private fun scheduleDailyReminder(
    context: Context,
    task: TaskItem,
    alarmManager: AlarmManager,
    hour: Int,
    minute: Int
) {
    val requestCode = getRequestCode(task.id, 0)
    val intent = createAlarmIntent(context, task)
    val pendingIntent = createPendingIntent(context, requestCode, intent)

    val calendar = Calendar.getInstance().apply {
        set(Calendar.HOUR_OF_DAY, hour)
        set(Calendar.MINUTE, minute)
        set(Calendar.SECOND, 0)
        set(Calendar.MILLISECOND, 0)

        // 如果今天的时间已过，安排到明天
        if (timeInMillis <= System.currentTimeMillis()) {
            add(Calendar.DAY_OF_YEAR, 1)
        }
    }

    setExactAlarm(alarmManager, calendar.timeInMillis, pendingIntent)
}
```

### 3.6 精确闹钟封装

```kotlin
private fun setExactAlarm(
    alarmManager: AlarmManager,
    triggerAtMillis: Long,
    pendingIntent: PendingIntent
) {
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
        alarmManager.setExactAndAllowWhileIdle(
            AlarmManager.RTC_WAKEUP,
            triggerAtMillis,
            pendingIntent
        )
    } else {
        alarmManager.setExact(
            AlarmManager.RTC_WAKEUP,
            triggerAtMillis,
            pendingIntent
        )
    }
}
```

### 3.7 Intent 和 PendingIntent 创建

```kotlin
private fun createAlarmIntent(context: Context, task: TaskItem): Intent {
    return Intent(context, AlarmReceiver::class.java).apply {
        putExtra(EXTRA_TASK_ID, task.id)
        putExtra(EXTRA_TASK_NAME, task.task_name)
        putExtra(EXTRA_TASK_ICON, task.icon)
    }
}

private fun createPendingIntent(
    context: Context,
    requestCode: Int,
    intent: Intent
): PendingIntent {
    return PendingIntent.getBroadcast(
        context,
        requestCode,
        intent,
        PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
    )
}
```

**Flag 说明：**
- `FLAG_UPDATE_CURRENT`：如果 PendingIntent 已存在，更新其 extras
- `FLAG_IMMUTABLE`：Android 12+ 必需，防止其他应用修改 Intent

### 3.8 闹钟接收器

```kotlin
class AlarmReceiver : BroadcastReceiver() {

    override fun onReceive(context: Context, intent: Intent) {
        val taskId = intent.getIntExtra(NotificationHelper.EXTRA_TASK_ID, 0)
        val taskName = intent.getStringExtra(NotificationHelper.EXTRA_TASK_NAME) ?: "任务提醒"
        val taskIcon = intent.getStringExtra(NotificationHelper.EXTRA_TASK_ICON) ?: "default"

        showNotification(context, taskId, taskName, taskIcon)
    }

    private fun showNotification(context: Context, taskId: Int, taskName: String, taskIcon: String) {
        // 1. 创建通知渠道（Android 8.0+）
        NotificationHelper.createNotificationChannel(context)

        // 2. 创建点击跳转 Intent
        val notificationIntent = Intent(context, MainActivity::class.java).apply {
            flags = Intent.FLAG_ACTIVITY_NEW_TASK or Intent.FLAG_ACTIVITY_CLEAR_TOP
        }

        val pendingIntent = PendingIntent.getActivity(
            context,
            taskId,
            notificationIntent,
            PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
        )

        // 3. 构建通知
        val iconResId = IconMapper.getResIdFromName(taskIcon)

        val notification = NotificationCompat.Builder(context, NotificationHelper.CHANNEL_ID)
            .setSmallIcon(iconResId)
            .setContentTitle("任务提醒")
            .setContentText("别忘了完成任务：$taskName")
            .setPriority(NotificationCompat.PRIORITY_HIGH)
            .setCategory(NotificationCompat.CATEGORY_REMINDER)
            .setAutoCancel(true)
            .setContentIntent(pendingIntent)
            .setDefaults(NotificationCompat.DEFAULT_ALL)
            .build()

        // 4. 显示通知
        val notificationManager = context.getSystemService(
            Context.NOTIFICATION_SERVICE
        ) as NotificationManager
        notificationManager.notify(taskId, notification)
    }
}
```

## 四、调度时机

### 4.1 何时调度提醒

**不是在任务创建时调度，而是在任务列表加载时调度。**

**HomeFragment.loadTasks() 中的调度逻辑：**

```kotlin
private fun loadTasks(date: LocalDate) {
    val dateStr = date.format(DateTimeFormatter.ISO_LOCAL_DATE)
    lifecycleScope.launch {
        loadingStateHelper.showLoading()
        try {
            val response = apiService.getTaskList(dateStr, currentCategoryFilter)
            if (response.isSuccessful && response.body()?.code == 200) {
                val tasks = response.body()?.data ?: emptyList()

                // 调度提醒
                tasks.forEach { task ->
                    if (task.reminder_enabled && !task.reminder_time.isNullOrEmpty()) {
                        // 先取消旧的提醒
                        NotificationHelper.cancelTaskReminder(requireContext(), task.id)
                        // 再设置新的提醒
                        NotificationHelper.scheduleTaskReminder(requireContext(), task)
                    }
                }

                updateTaskList(tasks)
                loadingStateHelper.showContent()
            }
        } catch (e: Exception) {
            // 错误处理...
        }
    }
}
```

### 4.2 为什么选择这个时机？

| 时机 | 优点 | 缺点 |
|------|------|------|
| 任务创建时 | 即时 | 修改任务后需要额外处理 |
| 任务列表加载时 | 自动刷新、修改后自动更新 | 依赖用户打开应用 |
| 定时同步 | 无需用户操作 | 复杂，需要后台服务 |

**选择「任务列表加载时」的原因：**

1. **自动刷新**：每次进入首页都会刷新，确保提醒是最新的
2. **修改自动更新**：修改任务后回到首页，先取消再重新设置
3. **设备重启恢复**：重启后首次进入首页会重新设置所有提醒
4. **实现简单**：无需后台服务、无需同步机制

### 4.3 调度流程

```
用户打开应用
    ↓
HomeFragment onCreateView
    ↓
loadTasks(selectedDate)
    ↓
请求 GET /task/list
    ↓
获取任务列表
    ↓
遍历任务：
    ├── reminder_enabled = true 且 reminder_time 非空
    │       ├── cancelTaskReminder()  // 取消旧提醒
    │       └── scheduleTaskReminder() // 设置新提醒
    │
    └── reminder_enabled = false
            └── cancelTaskReminder()  // 取消所有提醒
    ↓
更新 UI
```

## 六、注意事项

### 6.1 Android 版本兼容性

| Android 版本 | 特性 | 处理方式 |
|-------------|------|----------|
| < 8.0 | 无需通知渠道 | 跳过渠道创建 |
| >= 8.0 | 必须使用通知渠道 | createNotificationChannel() |
| < 6.0 | 标准闹钟即可 | setExact() |
| >= 6.0 | Doze 模式 | setExactAndAllowWhileIdle() |
| >= 12 | PendingIntent 必须可变性 | FLAG_IMMUTABLE |

### 6.2 权限问题

**精确闹钟权限：**
- Android 12+ 需要 `SCHEDULE_EXACT_ALARM`
- 部分 OEM 系统（如小米、华为）有额外的自启动权限限制

**建议：**
- 检测权限是否已授权
- 引导用户到设置页面开启

### 6.3 提醒持续生效问题

**当前实现的限制：**

由于 AlarmManager 的 `setExact` 系列方法是一次性的（非重复），每次触发后需要重新设置。

**问题：**
- 单次提醒：正常工作
- 每日/每周提醒：首次触发后不会自动设置下一次

**解决方案（当前做法）：**
- 依赖用户打开应用时重新调度
- 每次进入首页会刷新所有提醒

**更好的解决方案（未来优化）：**
- 使用 `setRepeating()`（不精确）
- 或在 AlarmReceiver 触发后重新设置下一次提醒

### 6.4 开机恢复

**问题：** 设备重启后，所有 Alarm 都会被清除。
