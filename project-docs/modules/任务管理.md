# 任务管理模块

## 一、功能概述

任务管理模块是 KeepPunch 的核心模块，负责：

- **任务创建**：创建新的习惯/目标任务
- **任务列表**：按日期和分类筛选显示任务
- **任务详情**：查看任务完整信息和统计
- **任务编辑**：修改任务属性
- **任务状态管理**：终止/恢复任务

### 2.3 核心数据模型

```sql
CREATE TABLE task (
  id INT AUTO_INCREMENT PRIMARY KEY,
  task_name VARCHAR(255) NOT NULL,
  icon VARCHAR(100) DEFAULT 'book',
  repeat_type INT DEFAULT 1,
  week_rule VARCHAR(50) DEFAULT '',
  target_count INT DEFAULT 0,
  status INT DEFAULT 1,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  reminder_time VARCHAR(10),
  reminder_enabled TINYINT DEFAULT 0,
  reminder_type INT DEFAULT 3,
  reminder_date VARCHAR(20),
  reminder_week_rule VARCHAR(50),
  category_id INT
);
```

### 2.4 重复类型设计

| repeat_type | 含义 | week_rule | target_count |
|------------|------|-----------|--------------|
| 1 | 按周重复 | 指定星期几（1-7） | 忽略 |
| 2 | 周目标 | 忽略 | 每周需要完成次数 |
| 3 | 月目标 | 忽略 | 每月需要完成次数 |

**示例：**
- 每周一三五重复：`repeat_type=1, week_rule="1,3,5"`
- 每周至少锻炼3次：`repeat_type=2, target_count=3`
- 每月至少跑步10次：`repeat_type=3, target_count=10`

### 2.5 任务列表筛选逻辑

```
输入：date=2026-06-09, category_id=1

步骤：
1. 基础过滤：status = 1（活跃状态）
2. 分类过滤：
   - category_id = null：不过滤
   - category_id = '0'：category_id IS NULL
   - category_id = '1'：category_id = 1
3. 周规则过滤（repeat_type=1）：
   - week_rule 为空：全部显示
   - week_rule = "1,3,5"：检查 2026-06-09 是周二（2），不包含，过滤掉
4. 目标计数过滤：
   - repeat_type=2：查询本周完成次数 < target_count
   - repeat_type=3：查询本月完成次数 < target_count
5. 关联查询：LEFT JOIN category 获取分类信息
6. 关联查询：LEFT JOIN record 判断 is_finished
```

### 2.6 软删除机制

**删除操作：**
```javascript
// 不是 DELETE，而是 UPDATE
const updateSql = `UPDATE task SET status = 2 WHERE id = ?`;
```

**恢复操作：**
```javascript
// 检查当前状态
if (task.status !== 2) {
    ctx.body = { code: 400, msg: '任务未处于终止状态' };
    return;
}
// 恢复
const updateSql = `UPDATE task SET status = 1 WHERE id = ?`;
```

**为什么用软删除？**
- 保留历史打卡记录（用于统计）
- 支持误删恢复
- 数据完整性

#### 核心逻辑

```kotlin
// 1. 初始化
override fun onCreateView(...): View {
    setupCalendar()           // 设置周历
    setupTaskList()           // 设置 RecyclerView
    loadCategories()          // 加载分类列表
    loadTasks(selectedDate)   // 加载当天任务
}

// 2. 日期切换
calendarView.dayBinder = object : WeekDayBinder<DayViewContainer> {
    override fun bind(container: DayViewContainer, data: WeekDay) {
        container.root.setOnClickListener {
            selectedDate = data.date
            loadTasks(data.date)  // 切换日期重新加载
        }
    }
}

// 3. 加载任务
private fun loadTasks(date: LocalDate) {
    val dateStr = date.format(DateTimeFormatter.ISO_LOCAL_DATE)
    lifecycleScope.launch {
        val response = apiService.getTaskList(dateStr, currentCategoryFilter)
        if (response.isSuccessful && response.body()?.code == 200) {
            val tasks = response.body()?.data ?: emptyList()
            
            // 调度提醒
            tasks.forEach { task ->
                if (task.reminder_enabled && !task.reminder_time.isNullOrEmpty()) {
                    NotificationHelper.scheduleTaskReminder(requireContext(), task)
                }
            }
            
            updateTaskList(tasks)
        }
    }
}

// 4. 分组显示
private fun updateTaskList(tasks: List<TaskItem>) {
    val incomplete = tasks.filter { !it.is_finished }
    val complete = tasks.filter { it.is_finished }
    
    binding.rvIncomplete.adapter = TaskAdapter(incomplete, ...)
    binding.rvCompleted.adapter = TaskAdapter(complete, ...)
}
```

### 5.1 日期处理

### 5.2 状态管理

| status | 说明 | 查询时是否显示 |
|--------|------|---------------|
| 1 | 活跃 | 是 |
| 2 | 终止 | 仅在"全部任务"中显示 |

### 5.3 提醒调度时机

提醒在**任务列表加载时**调度，而不是任务创建时：

```kotlin
// HomeFragment.loadTasks()
tasks.forEach { task ->
    if (task.reminder_enabled && !task.reminder_time.isNullOrEmpty()) {
        // 先取消旧的提醒
        NotificationHelper.cancelTaskReminder(requireContext(), task.id)
        // 再设置新的提醒
        NotificationHelper.scheduleTaskReminder(requireContext(), task)
    }
}
```

**原因：**
- 每次进入首页都会刷新，确保提醒是最新的
- 修改任务后回到首页会自动更新提醒
- 设备重启后首次进入首页会重新设置
