# 打卡功能模块

## 一、功能概述

打卡功能是 KeepPunch 的核心交互模块，负责：

- **每日打卡**：标记任务为已完成
- **取消打卡**：撤销当天的打卡操作
- **日期验证**：防止打卡未来日期
- **计数更新**：同步更新任务完成计数

```sql
CREATE TABLE record (
  id INT AUTO_INCREMENT PRIMARY KEY,
  task_id INT NOT NULL,
  record_date DATE NOT NULL,
  is_finish TINYINT DEFAULT 1,
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY unique_task_date (task_id, record_date)
);

CREATE TABLE task_count (
  id INT AUTO_INCREMENT PRIMARY KEY,
  task_id INT NOT NULL,
  year INT NOT NULL,
  month INT NOT NULL,
  week INT NOT NULL,
  total_count INT DEFAULT 0,
  UNIQUE KEY unique_task_period (task_id, year, month, week)
);
```

### 2.4 打卡事务处理

#### 为什么需要事务？

打卡操作涉及两个表的更新：
1. `record` 表：插入打卡记录
2. `task_count` 表：更新完成计数

如果没有事务保护，可能出现：
- 记录插入成功，但计数没更新 → 统计错误
- 计数更新了，但记录没插入 → 数据不一致

#### 事务实现

```javascript
router.post('/checkin', async (ctx) => {
    const { task_id, record_date } = ctx.request.body;
    
    // 1. 日期验证
    const targetDate = new Date(record_date);
    const today = new Date();
    today.setHours(0, 0, 0, 0);
    targetDate.setHours(0, 0, 0, 0);
    
    if (targetDate > today) {
        ctx.body = { code: 400, msg: '不能打卡未来日期' };
        return;
    }
    
    // 2. 获取独立连接（用于事务）
    const connection = await pool.getConnection();
    
    try {
        // 3. 开启事务
        await connection.beginTransaction();
        
        // 4. 检查是否已打卡（唯一约束防重复）
        const checkSql = `SELECT * FROM record 
                          WHERE task_id = ? AND record_date = ?`;
        const [existing] = await connection.query(checkSql, [task_id, record_date]);
        
        if (existing.length > 0) {
            await connection.rollback();
            ctx.body = { code: 400, msg: '今天已打卡' };
            return;
        }
        
        // 5. 插入打卡记录
        const insertSql = `
            INSERT INTO record (task_id, record_date, is_finish)
            VALUES (?, ?, 1)
        `;
        const [result] = await connection.query(insertSql, [task_id, record_date]);
        
        // 6. 计算年份、月份、周数
        const year = targetDate.getFullYear();
        const month = targetDate.getMonth() + 1;
        const firstDayOfYear = new Date(year, 0, 1);
        const pastDaysOfYear = (targetDate - firstDayOfYear) / 86400000;
        const weekOfYear = Math.ceil((pastDaysOfYear + firstDayOfYear.getDay() + 1) / 7);
        
        // 7. 更新/插入计数表（使用 ON DUPLICATE KEY UPDATE）
        const upsertWeekCountSql = `
            INSERT INTO task_count (task_id, year, month, week, total_count)
            VALUES (?, ?, ?, ?, 1)
            ON DUPLICATE KEY UPDATE total_count = total_count + 1
        `;
        await connection.query(upsertWeekCountSql, [task_id, year, month, weekOfYear]);
        
        // 8. 提交事务
        await connection.commit();
        
        ctx.body = {
            code: 200,
            msg: '打卡成功',
            data: { id: result.insertId }
        };
        
    } catch (err) {
        // 9. 出错回滚
        await connection.rollback();
        console.error('打卡失败:', err);
        ctx.body = { code: 500, msg: '数据库错误', error: err.message };
    } finally {
        // 10. 归还连接到池
        connection.release();
    }
});
```

### 2.5 取消打卡

```javascript
router.delete('/checkin', async (ctx) => {
    const { task_id, record_date } = ctx.request.body;
    
    const connection = await pool.getConnection();
    
    try {
        await connection.beginTransaction();
        
        // 1. 检查打卡记录是否存在
        const checkSql = `SELECT * FROM record 
                          WHERE task_id = ? AND record_date = ?`;
        const [existing] = await connection.query(checkSql, [task_id, record_date]);
        
        if (existing.length === 0) {
            await connection.rollback();
            ctx.body = { code: 404, msg: '该日期没有打卡记录' };
            return;
        }
        
        // 2. 删除打卡记录
        const deleteSql = `DELETE FROM record 
                           WHERE task_id = ? AND record_date = ?`;
        await connection.query(deleteSql, [task_id, record_date]);
        
        // 3. 减少计数（最小为 0）
        const updateCountSql = `
            UPDATE task_count 
            SET total_count = GREATEST(total_count - 1, 0)
            WHERE task_id = ? AND year = ? AND month = ? AND week = ?
        `;
        await connection.query(updateCountSql, [task_id, year, month, weekOfYear]);
        
        await connection.commit();
        ctx.body = { code: 200, msg: '取消打卡成功' };
        
    } catch (err) {
        await connection.rollback();
        ctx.body = { code: 500, msg: '数据库错误' };
    } finally {
        connection.release();
    }
});
```

### 2.6 计数表设计原理

#### 预聚合 vs 实时计算

**方案对比：**

| 方案 | 查询性能 | 写入性能 | 适用场景 |
|------|---------|---------|---------|
| 实时 COUNT() | O(n)，数据量大时慢 | 只写一张表 | 小数据量 |
| 预聚合计数表 | O(1)，直接查询 | 写两张表 | 读多写少 |

**本项目选择预聚合的原因：**
- 统计报表频繁查询
- 打卡操作相对较少
- 历史数据不再变化

#### ON DUPLICATE KEY UPDATE

```sql
INSERT INTO task_count (task_id, year, month, week, total_count)
VALUES (?, ?, ?, ?, 1)
ON DUPLICATE KEY UPDATE total_count = total_count + 1
```

**执行逻辑：**
1. 尝试插入新记录
2. 如果唯一约束冲突（该周已有记录）
3. 执行 UPDATE 部分，total_count + 1
4. 否则插入新记录，total_count = 1

**相当于：**
```sql
-- 检查是否存在
SELECT * FROM task_count WHERE task_id=? AND year=? AND month=? AND week=?

-- 如果存在
UPDATE task_count SET total_count = total_count + 1 WHERE ...

-- 如果不存在
INSERT INTO task_count (...) VALUES (...)
```

但用一条 SQL 完成，原子性更好。

#### HomeFragment 中的处理

```kotlin
private fun handleCheckin(task: TaskItem, isAlreadyChecked: Boolean) {
    // 1. 日期验证（前端也做一次验证）
    if (!isAlreadyChecked && selectedDate.isAfter(today)) {
        Toast.makeText(requireContext(), "不能打卡未来日期", Toast.LENGTH_SHORT).show()
        return
    }
    
    // 2. 构造请求
    val dateStr = selectedDate.format(DateTimeFormatter.ISO_LOCAL_DATE)
    val request = CheckinRequest(task.id, dateStr)
    
    lifecycleScope.launch {
        try {
            // 3. 调用 API
            val response = if (isAlreadyChecked) {
                apiService.cancelCheckin(request)
            } else {
                apiService.checkin(request)
            }
            
            // 4. 处理响应
            if (response.isSuccessful) {
                val body = response.body()
                if (body?.code == 200) {
                    // 5. 成功，刷新列表
                    loadTasks(selectedDate)
                } else {
                    // 6. 业务失败，显示后端消息
                    Toast.makeText(requireContext(), 
                        body?.msg ?: "操作失败", Toast.LENGTH_SHORT).show()
                }
            }
        } catch (e: Exception) {
            // 7. 网络异常
            Toast.makeText(requireContext(), "网络错误", Toast.LENGTH_SHORT).show()
        }
    }
}
```

## 五、关键设计点

### 5.1 双重验证

**前端验证：**
```kotlin
if (!isAlreadyChecked && selectedDate.isAfter(today)) {
    Toast.makeText(requireContext(), "不能打卡未来日期", Toast.LENGTH_SHORT).show()
    return
}
```

**后端验证：**
```javascript
if (targetDate > today) {
    ctx.body = { code: 400, msg: '不能打卡未来日期' };
    return;
}
```

**为什么都要验证？**
- 前端：提升用户体验，快速反馈
- 后端：安全保障，防止绕过前端直接调用 API

### 5.2 唯一约束防重复

```sql
UNIQUE KEY unique_task_date (task_id, record_date)
```

**作用：**
- 数据库层面保证不会重复打卡
- 即使并发请求，也只有一个成功
- 比先 SELECT 再 INSERT 更可靠

### 5.3 局部刷新优化

```kotlin
// 只刷新变化的日期单元格
calendarView.notifyDateChanged(oldDate)
calendarView.notifyDateChanged(newDate)

// 而不是刷新整个日历
// calendarView.notifyDataSetChanged()
```

**好处：**
- 性能更好
- 不会触发不必要的重绘

## 六、边界情况

### 6.1 跨天打卡

**场景：** 用户在 23:59 打卡，系统时间到 00:00

**处理：**
- 使用服务器时间作为标准
- 前端传入的日期字符串（yyyy-MM-dd）作为依据
- 后端验证：`record_date <= today`

### 6.2 并发打卡

**场景：** 用户快速双击打卡按钮

**处理：**
- 数据库唯一约束
- 第一个请求成功，第二个返回"今天已打卡"

### 6.3 打卡历史修改

**场景：** 用户想修改昨天的打卡记录

**当前支持：**
- 可以打卡/取消打卡今天及之前的日期
- 通过日历选择历史日期操作

**限制：**
- 不能打卡未来日期
